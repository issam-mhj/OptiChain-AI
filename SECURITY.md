# Security Policy

## Supported Versions

We release patches for security vulnerabilities for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability within OptiChain-AI, please send an email to security@optichain-ai.com. All security vulnerabilities will be promptly addressed.

Please include the following information:
- Type of vulnerability
- Full paths of affected source files
- Location of the affected code (tag/branch/commit)
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit it

## Security Best Practices

When deploying OptiChain-AI:
- Always use the latest version
- Keep all dependencies up to date
- Use environment variables for sensitive configuration
- Never commit credentials to version control
- Use secure connections (HTTPS/TLS) in production
- Implement proper authentication and authorization
- Regularly review access logs
- Follow the principle of least privilege

Thank you for helping keep OptiChain-AI secure!
